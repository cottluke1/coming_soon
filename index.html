Setup Instructions & Checklist

✅ 1. Google Sheet Setup

You only need a Google Sheet with headers:

Timestamp | First Name | Last Name | Email

✅ 2. Apps Script Setup

  • Inside the Google Sheet: Click Extensions > Apps Script.
  • Paste the doPost() script (provided in the other document).
  • Click Deploy > Manage Deployments > New deployment.
  • Choose Web App, set:
  •     Execute as: Me
  •     Access: Anyone
  • Click Deploy and authorize.
  • Copy the Web App URL and paste it into your SCRIPT_URL in the HTML.

✅ 3. Enable Access Settings

  • Make sure the Google Sheet is shared properly: You don’t need to make the sheet public.
  • Just ensure that the Apps Script has permission to write to it.
  • If your form doesn't work and you get "Permission denied", go back to the Apps Script deployment and make sure “Who has access” = Anyone.

✅ 4. No Backend Required

Because your HTML form uses fetch() to send data to the Web App, you do not need Node.js, PHP, or any server — it's 100% front-end compatible. You’re good using GitHub Pages or any static site hosting.

✅ 5. Files You Need

  • Only these two:
  •     Your HTML file (this page)
  •     The Google Apps Script deployed and connected to your Sheet
  • You don’t need to host anything else unless you're adding analytics, logging, or email confirmations.

✅ Final Checklist (Hosting on GitHub Pages)

  • Google Sheet created with correct headers✅
  • Apps Script pasted and deployed as Web App✅
  • Web App URL copied and inserted into HTML✅
  • GitHub Pages enabled for your repo✅
  • HTML hosted and tested live✅
